Suite des docs 26/27. Cinq irritants relevés par Clavel après test réel,
cinq corrections livrées.

## 1. Double login app → WebView : RÉGLÉ (SSO par ticket signé)

Le pire irritant (« l'user se dit je me co deux fois »).
- **Serveur** : `Api\PirbSsoTicketController` (`POST /api/pirb/sso/ticket`,
  Bearer → ticket HMAC `userId.exp.signature`, 90 s, secret =
  `%kernel.secret%`, AUCUNE table) + `Pirb\SsoController`
  (`GET /sso/pirb?ticket=…&cible=…`, PUBLIC sur pirb.* — voir la nouvelle
  règle access_control — vérifie la signature, `Security::login(...,
  'form_login', 'pirb')` pose le cookie, redirige vers la cible locale
  uniquement, anti open-redirect).
- **App** : `urlWebConnectee(cible)` dans le contrat (Api : ticket → URL
  SSO ; échec → URL brute, dégradation douce). Branché dans `app/web.tsx`,
  LE point d'entrée unique des pages web → tous les appelants (menu,
  shot chart, séances…) profitent du SSO sans modification.
- ⚠️ Déploiement : cache:clear obligatoire (nouvelle route + access_control).

## 2. Sélecteur de saison disparu des Stats : RÉGLÉ

Le sélecteur existait mais `getSaisons()` échouait en silence →
`saisons.length > 1` cachait les chips. Fix : repli LOCAL dans
`ApiPirbDataService.getSaisons()` (même règle que le serveur : bascule au
1er juillet, jamais de future, jusqu'à 2023-2024). Le menu existe
TOUJOURS, le serveur reste la référence quand il répond.
À creuser quand même côté serveur : pourquoi /api/pirb/saisons échouait
(vérifier en prod une fois déployé).

## 3. Équipes du club cliquables : FAIT

`club.tsx` : chaque ligne d'équipe → **nouvel écran `app/equipe.tsx`**
(joueuses de l'équipe via la commu filtrée, bouton Suivre optimiste).
L'équipe devient une porte d'entrée vers la commu. Zéro endpoint ajouté.
Note dette : la ligne joueuse+Suivre existe maintenant en 3 exemplaires
(social, equipe, CommuCard) → extraire un composant partagé au prochain
passage (règle des trois atteinte).

## 4. Carte illisible sur iPad : RÉGLÉ

`carte.tsx` : hauteur = min(52 % de l'écran, 460 px) via
`useWindowDimensions` (avant : aspectRatio 1/1.1 → plus haute que l'écran
iPad, la fiche du club tapé était invisible sous le pli).

## 5. Explorateur de badges + manifeste « les points ne font pas tout » : FAIT

- **Nouvel écran `app/badges.tsx`** : tous les badges (débloqués +
  verrouillés grisés), groupés par axe dans un ordre qui EST le message —
  Régularité, Esprit club, Performance, Parcours. Manifeste en tête :
  « être présente, aider à la table de marque… ça compte autant que les
  paniers ». Compteurs par axe, débloqués d'abord.
- **Profil** : compteur badges cliquable → /badges ; la vitrine ne montre
  plus QUE les débloqués (la fierté) + bouton « Explorer tous les badges ».
